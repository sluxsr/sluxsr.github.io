阅读笔记：Exploring Logic Optimizations with Reinforcement Learning and Graph Convolutional Network

· 2021-12-31 ·

今天准备阅读以下这篇文章： Exploring Logic Optimizations with Reinforcement Learning and Graph Convolutional Network

摘要：组合电路的Logic synthesis指的是找到Boolean Logic Functions的最小等效表示。常用做法是用Standardized logic networks来表示Boolean logic，例如反相图 (AIG)，并在图上迭代地执行逻辑最小化操作。尽管已经有很多对不同逻辑表示和运算的研究，但这些运算的顺序往往都是由常识（启发式算法）决定的。我们提出了解决Logic synthesis问题的马尔可夫决策过程 (MDP) ，并且结合图卷积网络和强化学习 (RL) 算法，以探索解决方案搜索空间。实验结果表明，在相同的序列长度和动作空间下，所提出的方法优于众所周知的逻辑综合启发式算法。

Logic synthesis发生于将 “functional descriptive language” (RTL, register-transfer level language) 翻译为 “logic gate-level descriptions” 的过程中，即，要将hardware logic翻译为诸如NAND, NOR, XOR, INV之类的gates。Logic synthesis的目的是减少映射的devices（gates）的数量、减少latency，提升运行速度。优化通常发生于Logic function level，这个阶段通常由logic graphs来表示。在不改变logic function的情况下，logic graph中一些诸如balance, rewrite, refactor之类的操作，可以被优化为其他操作。优化目标可以是：更少的nodes或者较浅的最大logic graph depth。

Preliminary

Logic Networks/AIG： 是combinational logic functions的图表示，每个node代表一个基本逻辑门，nodes之间的连接代表下游logic paths。其中and-inverter graph只由AND和NOT gates组成。

Logic Synthesis： 在将硬件语言翻译为gate-level的同时优化area, delay, power。要想scale to large graphs需要算法能够根据图的一些heuristics局部优化子图，然后反复执行这些操作，直到不能继续优化。

Markov Decision Process: formulates the dicrete-time stochastic control process. 下一步action的probability distribution只取决于当前state。

本文算法

MDP Formulation：Logic synthesis本来就是一个序列化的过程。

  1. State space：首先state就是跟之前的一些工作类似，是用current logic graph来表示。比如，
    • 现在graph中有多少nodes，多深的depth，
    • 上一action之前有少nodes和depth，
    • 上个action是什么，
    • 前三个action的sum，并做归一化，
    • 现在在执行第几个action，并normalized by 18（我猜这个18是由作者设定的每个episode可以包含的最多steps的数量）
    • AIG graph representation using GNN，细节将在后文展开
  2. Action space：用的就是resyn2的action space，就5种actions。
动作缩写 命令
b Balance: 平衡当前网络
rw Rewrite: 重写AIG
rf Refactor: refactor AIG
rwz Rewrite with zero-cost replacements
rfz Refactor with zero-cost replacements
  1. Rewards: 用nodes和depth的gain（即通过优化减少了多少nodes和depth）减去一个baseline。baseline通过run两次resyn2，并取平均值得到。

RL算法：

用的是REINFORCE algorithm，用policy 学习每个action的value，从而得到actions的probability distribution，最终选择action。MCTS的随机性令RL agent得以在整个action space中探索。本文使用一个state-value function parameterization 来估算state的value，这部分内容在后文有详细讲解。

GCN (Graph Concolutional Network)：
本文采用的是传统的GCN，其中作为input的node embedding包含了每个node的logic node type和PI/PO type（即一个有六种可能的one-hot vector，(1)constant one, (2)primary output, (3)primary input, (4)no inverter, (5)one inverter, (6)two inverters），经过几层图卷积层，得到的新node embedding再取全部node的平均，从而得到graph embedding。

实验部分

因为模型非常小，所以是在普通PC上用CPU训练的。

本文用他们的模型run 100次（每次包含200 episodes）和run两次的resyn2在一个叫i10的benchmark上作对比。我们上文已经提到，他们的reward baseline就是通过run两次resyn2得到的，所以黑线就是resyn2，蓝线是本文的模型，可以看到本文的模型对resyn2是有一些提升的。

接下来，本文又随机从候选集中选了8个相对较大的combinational circuit benchmarks，在本文介绍的模型中对每个benchmark run 10次，得到两组结果RL-1和RL-2，这两组的区别在于一个基于node数量优化，另一个基于depth优化。Baselines是：run一次resyn2 （resyn2-1）、run两次resyn2 （resyn2-2）、至少run五次resyn2直至结果不变（（resyn2-））。结果如下：
